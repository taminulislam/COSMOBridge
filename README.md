# COSMOBridge

**Bridging 2D Molecular Graphs and 3D Electrostatic Surfaces for Ionic Liquid Thermodynamic Property Prediction**

COSMOBridge is a multimodal architecture that combines Chemprop's directed
message-passing neural network (2D molecular graph) with a PointNet encoder over
3D COSMO electrostatic surfaces, blended through a temperature-conditioned
bilinear fusion and learned **per-property routing gates**. With only 7 trainable
parameters blending two frozen specialist paths, the minimal model (v3) matches a
strong graph baseline on average across 7 thermodynamic properties while
delivering significant, physically-localized gains on the surface-dependent
activity coefficients (γ₁, γ₂). An extended variant (v4) adds a per-molecule
router over three frozen paths.

This repository contains the **model and pipeline source code** accompanying the
manuscript (under review). It does **not** include the experimental dataset,
trained checkpoints, or raw logs — see [Data & model availability](#data--model-availability).

## Repository layout

```
src/
  data/         Dataset assembly, graph builder, COSMO preprocessing
  models/
    graph/      Chemprop D-MPNN, GNN/GAT, atom-surface D-MPNN
    pointcloud/ PointNet, DGCNN surface encoders
    fusion/     COSMOBridge (dual-path gates), GBH bilinear fusion,
                cross-attention, FiLM, MoE, fusion variants
    tabular/    Descriptor baselines
    vision/     ViT (COSMO-image experiments)
    generative/ Candidate-generation experiments (VAE/CVAE/GAN)
  training/     Trainer, losses, metrics
  utils/        Config / seeding
scripts/
  pipeline/     COSMO surface-generation pipeline
                (geometry opt → Psi4/NWChem COSMO → surface descriptors → point clouds)
  cosmobridge_multiseed.py     v3 multi-seed training/eval
  predict_cosmobridge_v3.py    virtual screening with the v3 model
  rank_ils_desirability.py     Derringer–Suich desirability ranking
  cosmo_sac_baseline.py        physics (COSMO-SAC) baseline
  revision_*.py                analyses for the revised manuscript
  ...           architecture-search variants, figure generation
cosmobridge_v4/
  models/ scripts/ configs/    per-molecule routing extension (v4)
configs/        YAML experiment configs
requirements.txt
```

## Installation

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

Key dependencies: PyTorch ≥ 2.0, PyTorch Geometric, RDKit, Chemprop, scikit-learn,
NumPy/Pandas, Matplotlib. The COSMO surface-generation pipeline additionally
requires Psi4 and/or NWChem and xtb (GFN2-xTB) for geometry optimization.

## Data & model availability

To respect collaborator data-sharing terms, this repository ships **code only**:

- **Experimental dataset** (activity coefficients and excess properties of the
  protic ionic liquids) — not included. Configs reference a local
  `Activity coeff and Excess Enthalpy for Imaging.xlsx`; point `data.excel_path`
  at your own copy. Available from the authors on reasonable request.
- **Trained checkpoints** — not included (regenerate via the training scripts).
- **COSMO surfaces / point clouds, logs, results, figures** — not included
  (regenerated by `scripts/pipeline/` and the figure scripts).

## Reproducing the main result

1. Generate (or obtain) COSMO surfaces and point clouds: `scripts/pipeline/step1…step5`.
2. Train the frozen specialist paths, then the v3 gates: `scripts/cosmobridge_multiseed.py`.
3. (Optional) Train the v4 per-molecule router: `cosmobridge_v4/scripts/`.
4. Virtual screen + rank candidates: `scripts/predict_cosmobridge_v3.py`,
   `scripts/rank_ils_desirability.py`.
